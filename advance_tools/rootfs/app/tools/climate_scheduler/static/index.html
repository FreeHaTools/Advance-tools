How Climate Scheduler works

What a schedule does

A schedule is a weekly map of target temperatures for one thermostat (climate entity). You paint blocks like “Mon–Fri 06:30–08:30 → 21°” on a grid. Every 60 seconds the add-on checks each enabled schedule: if the current time is inside a block it makes sure the thermostat is on and set to the block's temperature; outside all blocks it does what you chose — turn off, hold a fallback temperature, or nothing.

Important to know

  • The add-on must stay running. Schedules are enforced by this add-on itself — no Home Assistant automations are created. If the add-on stops, thermostats keep whatever setting they had last.
  • Times use the Home Assistant server's local time zone.
  • The scheduler only calls Home Assistant when something actually differs (target temperature off by more than 0.05°, or the unit is off during a block), so it never spams your devices.
  • If you change the thermostat by hand during a block, the scheduler will set it back within a minute. Pause the schedule (toggle on the card) when you want manual control.

Examples

Workday heating
  Mon–Fri: 06:30–08:30 → 21°, 17:00–22:00 → 21.5°
  Sat–Sun: 08:00–23:00 → 21°
  Outside blocks: turn off

Night setback (never fully off)
  Every day: 07:00–23:00 → 21°
  Outside blocks: keep at fallback 17°

Tips

  • Design Monday first, then use “Copy Monday to” to fill the week and tweak the weekend.
  • The temperature chips follow your thermostat's step size (usually 0.5°); values outside its min/max range are rejected.
  • Don't run a second schedule against the same thermostat — and disable any schedule built into the device itself, or the two will fight each other.
  • “Do nothing” outside blocks is great when another system (or a human) should stay in charge between blocks.
