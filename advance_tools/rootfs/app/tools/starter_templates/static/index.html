How Starter Templates works

1 · Pick a layout

Every template is a real, finished wall-tablet design — group boxes, a clock, dimmer rows, a thermostat dial, status lights. What you see on the card is what gets built.

2 · We match it to your home

A template does not hard-code entity ids. It describes slots — "the living room light", "the front door contact". We score every entity you own against each slot using its domain, its device class, its name, and the Home Assistant area it belongs to. Entities that are unavailable, unknown or left over from a removed device are pushed to the bottom, so a dead entity never wins.

3 · You review, then Apply

Every match is shown with its live state. Change any of them with the picker. Slots marked required must be filled before Apply unlocks; optional slots you leave empty simply remove their widget from the layout instead of leaving a broken tile behind.

Nothing is overwritten

  • Applying a template creates a new dashboard at the address you choose. Your existing dashboards are never modified.
  • If the address is already in use we stop and ask first.
  • Nothing in Home Assistant changes — no entities, no automations, no settings. This tool only writes a dashboard.
  • The new dashboard's entity allowlist is set to exactly the entities you mapped, so the tablet can reach those and nothing else.

Run it as often as you like

The result is a completely ordinary Dashboard Maker dashboard. Open it in the visual designer and move, restyle or delete anything. Want a different look? Build a second one from another template and keep whichever you prefer.

Adding your own templates

Templates are plain JSON files in the tool's templates/ folder. Drop one in and it appears in the gallery — no restart, no code.
