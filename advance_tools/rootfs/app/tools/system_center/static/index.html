How System Center works

The support bundle is safe to share

It is built for exactly one purpose: giving a stranger enough information to debug your installation without giving them anything that could get into it. Every value goes through one redaction filter that is deny-by-default — if a field name contains password, pin, hash, token, secret, key, credential, auth, cookie, api, webhook, chat_id, email or phone, the value is replaced with <redacted> and so is everything nested inside it.

On top of that, any string that simply looks like a credential is removed whatever its field is called: long hex or base64 strings, Bearer … headers, and URLs that carry a user:password@. The filter deliberately errs on the side of removing too much.

The bundle also leaves out your dashboard designs — only names, sizes, widget counts and skin names are included, so your layout and your personal room names are not published either. Before you download it, the page shows every file that will be inside and how many values were redacted. You can always open the zip and read report.md yourself.

The export is your own data and is NOT redacted

Export exists to move or back up a setup, so it has to contain the real values — your alarm configuration, your notification targets, your dashboards. Treat the file like a password manager export: it belongs on your own disk, not in a public issue tracker.

Password hashes are left out unless you explicitly tick the credentials option, because that is the one part of an export that is worth attacking offline. Without them, accounts arrive on the new system and you set fresh passwords there.

Never send an export when someone asks for "your config" — send a support bundle instead. That is what it is for.

Imports always make a rollback point first

Before a single file is written, System Center copies panel.json and every configuration file and dashboard it is about to touch into a timestamped restore point under the add-on's data folder. The last five are kept. The Rollback tab restores any of them, and takes a snapshot of the current state before it does, so you can step back and forth.

You choose per item what to import, and whether existing items are merged (your values kept, imported values laid on top) or replaced (wiped and rewritten). Neither mode deletes anything you did not tick.

You cannot lock yourself out

The account you are signed in with is protected. An import can never delete it, never demote it out of admin, and never change its password — if the file would do any of those, the import is refused or that part is skipped, and you are told why. The same check runs again after everything is written; if it fails, the whole import is rolled back automatically.

Malicious archives

Uploaded zips are checked before anything is unpacked: absolute paths, .. traversal and symlinks are rejected, the number of entries and the total unpacked size are capped, and suspicious compression ratios are refused. An archive whose format version this add-on does not understand is rejected outright, and one from a newer version of Advance Tools produces a warning.
