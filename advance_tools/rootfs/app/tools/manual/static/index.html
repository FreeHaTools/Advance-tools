🚀Getting started

Advance Tools is a toolbox that runs next to Home Assistant as an add-on. It has its own login system (independent from Home Assistant users) and a growing set of tools, each designed to make one part of your smart home easy and visual.

First run — the setup wizard

  1. Open http://YOUR_HA_IP:8234/ (or your custom domain), or click Advance Tools in the Home Assistant sidebar.
  2. A brand-new install opens the setup wizard: choose your admin password, optionally pick a starter layout that builds your first dashboard automatically, and you are done.
  3. After that you sign in normally with the password you chose.
There is no default password any more. If you are upgrading from an older version and still use admin / admin, you will be asked to change it the next time you sign in — this panel can unlock doors and disarm your alarm, so treat that password like a house key.

Sign-in protection

  • Passwords must be at least 8 characters and not an obvious one.
  • After 5 failed attempts for the same account, sign-in locks for a short time that grows with each further failure (30 seconds up to 15 minutes). It never locks permanently.
  • Hub → Security log shows recent sign-ins, failures, lockouts and password changes since the add-on last started.
  • Anyone can change their own password without needing an admin.

The Hub

After login you land on the Hub — the overview page with system stats (entities, automations, uptime) and a card for every installed tool. Tools open inside the hub; the ✕ Close tool button (top-right) or the tool's own ← Hub button brings you back.

Users & access

Hub → Users: create accounts for family members or wall tablets. Only admin users can open the hub and tools; normal users can only open the dashboards you grant them (see Dashboard Maker). Tablet sessions last 365 days so kiosks stay logged in.

Every tool speaks beginner

  • ? bubbles — hover any small ? circle for a plain explanation with a concrete example.
  • 📖 Learn buttons — most editors have a Learn panel describing what the thing is and when to use it.
  • Empty states — a fresh tool always tells you what your first step should be.

📈History Explorer

See what actually happened in your home. Pick up to six entities, choose a time range, and read it off a chart instead of guessing.

What you get

  • Line chart for anything numeric — temperature, power, humidity — with a hover crosshair showing every series at that instant.
  • State timeline for on/off things — doors, lights, locks, motion. A coloured band per entity: this is how you read "when was the front door open".
  • Statistics table: minimum, maximum, average, change over the period, and for on/off entities the total on-time and how many times it switched.
  • CSV download of exactly what you are looking at.

Ranges and resolution

Quick buttons (last hour, today, 24 hours, 7 / 30 days, this month) or a custom from/to. Short ranges use your recorder's raw history; longer ones use hourly or daily statistics so a month of data doesn't choke the tablet. The tool always tells you which one it used.

Averages are time-weighted: a value that held for six hours counts six times more than one that lasted a minute. A plain average of samples would quietly mislead you.
Gaps in a line are real — the entity had no data then (offline, restarted, or excluded from your recorder). They are never drawn as zero. How far back you can look is set by your Home Assistant recorder retention, not by this tool.

🧰System Center

Housekeeping: get help when something is wrong, and take your setup with you.

Support bundle

One button collects everything someone would need to help you — add-on and Home Assistant versions, uptime, the add-on log, which tools are loaded, and every tool's configuration — into a single zip.

  • Secrets are stripped automatically: passwords, PINs, tokens, API keys, hashes, webhook URLs, e-mail addresses and chat IDs are replaced with <redacted>.
  • Before you download, the tool shows you the file list and how many values were removed — so you can share it without worrying.
  • report.md inside the zip also lists problems it spotted by itself, like Home Assistant not being connected.

Export & import

  1. Export your dashboards and every tool's settings into one file. Use it to move to a new Home Assistant machine, snapshot before a big change, or share a setup with a friend. Account passwords are left out unless you deliberately tick the box.
  2. Import shows you what the file contains and what it would overwrite, and only changes what you tick.
  3. Every import writes a rollback point first. The last five are kept on the Rollback tab, one click to undo.
The export is your own data, not redacted — treat it like a backup of your passwords drawer. The support bundle is the one that is safe to hand to a stranger.
An import can never delete or demote the admin account you are signed in with — that check cannot be turned off.

📦Starter Templates

The fastest way to a finished dashboard. Pick a ready-made layout and it is matched against the devices you actually own — you review the choices, press Apply, and the dashboard exists.

The templates

  • 🏠 Family Home — lights for the main rooms, thermostat, door and motion status, a family list and a clock. The everyday wall tablet.
  • 🏢 Apartment — a denser layout for a smaller home, with one climate control and an energy summary.
  • 🔐 Security Tablet — portrait layout built around the security keypad, with door/window/motion tiles and a camera. Mount it by the front door.
  • 🏝️ Vacation Rental — guest-safe controls only, plus Wi-Fi and check-out information panels.

How the matching works

  1. Each template describes slots ("the living-room light", "the front door sensor") rather than fixed entities.
  2. Every slot is scored against your entities by domain, device class, area and name. Unavailable or dead entities are never chosen.
  3. The review screen shows what was matched and lets you change any of it with a searchable picker. Required slots must be filled before Apply becomes available.
  4. Apply creates a new dashboard — nothing existing is ever overwritten, and the dashboard's entity allowlist is set to exactly the entities you mapped.

Afterwards it is an ordinary dashboard: open it in Dashboard Maker and move, restyle or extend anything.

Adding your own template is just a JSON file dropped into the tool's templates/ folder — no code needed.

📊Dashboard Maker

Design beautiful control dashboards for wall tablets — no coding needed. Each dashboard is pixel-perfect at your tablet's exact resolution, protected by its own user access list, and locked into kiosk mode.

How to build one

  1. Open Dashboard Maker → ＋ New dashboard, give it a name (e.g. bedroom).
  2. Open the Designer. Pick your tablet preset in Page setup (Fire HD, iPad, Galaxy Tab…) so the canvas matches the screen.
  3. Add elements from the gallery: switches, buttons, sensors, climate dials, lights, cameras, charts, vacuum cards, media, navigation buttons and more — 90+ skins in themed packs.
  4. Drag, resize and snap elements anywhere (freeform canvas), bind each one to an entity with the searchable picker.
  5. Save, then grant users access and open /d/<name>/ on the tablet.

Good to know

  • Entity allowlist — every dashboard can only see and control the entities you list for it. Guests can't poke around.
  • Skins & packs — import extra skin packs (JSON) or build your own; the admin page has full pack-building docs.
  • Kiosk behavior — no exit without logout; enable the hidden logout gesture in Page setup (tap a corner N times).
  • Scaling modes — Fit (letterbox), Fill (crop), Stretch.
  • Home Life widgets — Family Notes, Shopping List, Energy Summary and Intercom bring the other tools onto your dashboards (see each tool's section).
Design at the tablet's exact preset resolution and keep scaling on Fit — everything will land pixel-perfect.

⚙️Automation Maker

Browse, run and build Home Assistant automations with a visual WHEN / AND IF / THEN builder. Everything you create is a real HA automation — it also shows up in Home Assistant's own editor.

The list

  • Every automation with its state, last trigger time and a toggle to enable/disable it. ▶ Run triggers it manually. ⧉ duplicates it as a starting point.
  • YAML · read-only automations live in your YAML files. You can still run/toggle them — and the 🔁 Convert button clones one into an editable copy and disables the YAML original (your files are never modified).

The builder

  1. WHEN (orange) — pick a trigger chip: state change, numeric threshold, time, sun, repeating interval, template.
  2. AND IF (blue) — optional conditions: state, time window with a weekday picker, template.
  3. THEN (green) — actions: call any service with the searchable service + entity pickers, delays, wait-until, stop.

Each block shows a live plain-English summary, and the side panel shows the exact YAML that will be saved as you type.

Motion light: WHEN binary_sensor.hallway_motion turns on → AND IF time is after 21:00 → THEN turn on light.hallway at 30% and start a timer to switch it off.

🧩Helper Maker

Create, edit and live-control every Home Assistant helper in one place — 21 types, each with a Learn panel and an example automation.

Basic helpers (instant, fully editable)

Toggle, Button, Number, Text, Dropdown, Date/Time, Counter, Timer, and Schedule (with a visual weekly block editor). Every card has inline quick controls: flip the toggle, drag the slider, start/pause the timer with a live countdown, bump the counter — without leaving the page.

Advanced helpers (combine & transform)

Group, Template, Threshold, Times of the Day, Trend, Min/Max, Statistics, Derivative, Integral, Utility Meter, Random, Switch-as-X — created through Home Assistant's own config flows rendered inside the tool.

Superpowers

  • 🔎 Usage finder — see every automation, script and scene that references a helper; the delete dialog warns you if it's still used.
  • ☑ Bulk mode — multi-select for bulk delete, plus JSON export/import of basic helpers (backups, copying between installs).
  • ⧉ Duplicate, copy entity id, type filter chips, search (/), sort by name/type/last-changed.
Guest mode: create a Toggle called “Guest mode”, then in Automation Maker add an AND IF condition “guest mode is off” to your noisy automations.

🩺Entity Doctor

A full health check-up for every entity in your system, with safe one-click fixes. Run it monthly — smart homes collect clutter.

What the scan finds

  • Unavailable / unknown entities (dead devices, broken integrations) — with a 🔄 reload button per integration.
  • Low batteries (threshold adjustable) and stale entities that haven't changed in N days.
  • Orphaned registry leftovers and duplicate names, plus info lists for disabled/hidden/no-area entities.
  • A 0–100 health score — category boxes double as filters.

Fixing things

  • Per entity: rename (name and/or entity id), hide, disable, remove (server-guarded — only true orphans can be deleted).
  • 🔧 Repair per category builds a preview plan: a checklist of proposed actions with reasons. Nothing runs until you press Apply; risky items start unchecked.
  • Usage finder before you touch anything — see what references the entity.
Duplicate names are usually harmless clones from one device (e.g. a switch + its firmware-update entity). The repair plan renames them by area/device so voice assistants stop mixing them up.

🎬Scene Maker

Build scenes by snapshotting reality: set the room exactly how you like it, capture it, done.

Workflow

  1. Set up the room physically — dim the lights, set the thermostat.
  2. 📸 Snapshot → pick the area → untick anything you don't want captured → capture. Lights keep brightness & color, climate keeps target temps, covers keep positions, helpers keep values.
  3. 🧪 Test now applies the scene live without saving — walk around, adjust, test again. Optional transition seconds for smooth light fades.
  4. Save. The scene appears in HA too, and any dashboard button or automation can activate it.

Every captured entity is an editable card, and a live YAML preview shows exactly what will be stored. YAML-defined scenes are listed activate-only, like in Automation Maker.

“Movie night”: lights at 15% warm white, TV on, blinds down. Snapshot the living room once — activate forever.

🚨Alert Maker

Watchdog rules that ping your phone when something needs attention. Each rule becomes a real HA automation, generated and kept up to date for you.

The 5 alert types

  • Left open / on — a door, window, lock, valve or switch stays open longer than X minutes.
  • Low battery — battery level drops below your threshold.
  • Went offline — a device becomes unavailable for a while.
  • Value too high / low — any numeric sensor crosses a limit (temperature, humidity, soil moisture…).
  • Enters a state — free-form: any entity, any state, optional duration.

Details that matter

  • One rule can watch many entities; the notification names the exact one that caused it.
  • Targets: any notify.* service (phones with the companion app) and/or a persistent notification in HA's 🔔 drawer.
  • 🧪 Test sends a real test notification so you know delivery works. Pause/resume toggles the automation.
“Freezer door left open > 5 min → notify everyone” and “any battery < 15% → persistent notification” cover 90% of households.

🔔Notify Hub

One place for every notification: where messages go (channels), what causes them (rules), and a Telegram bot that talks back.

1 — Channels

  • Telegram — a chat or group your own bot messages.
  • Home Assistant notify — the companion app on a phone, email, or any other notify.* service HA already has.
  • Home Assistant panel — a persistent notification in the sidebar.
  • Webhook — POST to any URL, with ready-made Discord and Slack body formats.

Every channel has a 🧪 Send a test button, so you find out it works before you rely on it.

2 — Rules

  • A device does something — a sensor reaches a state, a number crosses a limit, or anything changes. Optional "and stays that way for X minutes" and a time window. This type becomes a real HA automation, so you can inspect it under Settings → Automations (but edit it here).
  • Home Assistant problems — errors and warnings from the log, and restarts. An ignore list silences integrations you can't fix.
  • Dead or flat devices — anything unavailable longer than X minutes, plus low batteries. Each device is reported once; if it recovers and dies again, you hear about it again.
  • Scheduled digest — a summary at a time you pick: lights on, doors open, unlocked locks, climate, weak batteries, unavailable entities.

Each rule picks its own channels, can be marked urgent (ignores mute and quiet hours), and can have a cooldown so a chatty sensor can't spam you.

3 — Quiet hours and mute

Quiet hours hold non-urgent notifications between two times. The 🔕 Mute button in the header pauses everything for an hour. Both are recorded in the history, so nothing silently disappears.

4 — The Telegram bot

  1. Message @BotFather in Telegram, send /newbot, answer two questions.
  2. Paste the token it gives you into Notify Hub → Telegram bot → Save.
  3. Open a chat with your new bot, send /start, then press Find my chats to pick up your chat ID.
  4. Put that ID in Who may command the bot, and create a Telegram channel with the same ID to receive notifications there.

Commands: /status (how the house is right now), /rules (tap a rule to switch it on or off), /control (buttons for the devices you allowed), /find kitchen, /mute 2h, /unmute, /log, /help.

Only chat IDs on your allow-list can command the bot, and only the entities you tick under Quick controls can be switched from Telegram. Anyone else who messages the bot is told their own chat ID and nothing else.
“Front door becomes on between 23:00 and 06:00 → Telegram + phone, urgent” plus a 07:30 daily digest covers most homes. Add a dead-device rule with a 60-minute threshold and you'll know about a flat sensor battery before it matters.

💾Backup Manager

See, create, schedule and download Home Assistant backups without leaving the toolbox.

  • List — type (full/partial), size, age, password protection, what's inside; 🔎 details shows every included add-on + version.
  • Create — full, or partial with a searchable add-on/folder picker; optional password; optional “exclude history database” for much smaller files. Runs in the background with a progress banner.
  • Schedule — pick a time and weekdays; automatic cleanup keeps only the last N scheduled backups (manual backups are never touched).
  • ⬇ Download any backup as a .tar (works for multi-GB files), 🗑 delete with confirmation.
Schedule a weekly full backup at 03:00 and set cleanup to keep 4 — a month of safety for a few GB.

🌡️Climate Scheduler

Weekly temperature schedules for your thermostats — drawn, not typed. The add-on applies them every minute; no automations to maintain.

How to use it

  1. ＋ New Schedule → pick a climate entity.
  2. Paint the week: pick a temperature chip (colored cold-blue → warm-red), then click-drag across the 7-day grid to draw heating blocks. Click a block to fine-tune exact times and temperature.
  3. Choose what happens outside blocks: turn off, hold a fallback temperature (night setback), or do nothing.
  4. “Copy Monday to weekdays/weekend/all” fills the week fast. A live sentence under the grid describes the schedule in plain English.
  5. Enable it. The card shows what's happening right now and an activity log shows every change the scheduler made.
The add-on must be running for schedules to apply (it is, as long as HA is up). Manual thermostat changes inside a block are re-applied within a minute — pause the schedule for manual control.
Workdays: 06:30–08:30 → 21°, 17:00–22:30 → 21.5°, otherwise hold 17°. Weekend: one long 08:00–23:00 → 21° block.

⚡Energy Center

Where your electricity actually goes — consumption and cost per device, built on Home Assistant's long-term statistics.

Setup (once)

  1. The first-run wizard lists every energy sensor (kWh) it can find — tick the ones to track (smart plugs with energy monitoring, your mains meter…).
  2. Enter your electricity price per kWh and currency.

The dashboard

  • Ranges: Today / Yesterday / This week / This month.
  • Stat cards: total kWh, total cost, hungriest device, peak hour or average per day.
  • Consumption bar chart over the range + Top consumers ranking; click a device to drill into its own chart and share %.
A smart plug without an energy sensor? Create an Integral (Riemann) helper in Helper Maker to turn its power (W) into energy (kWh), then track that here.

📺 On your dashboards

The Energy Summary widget (designer → Home Life) shows today's kWh + cost, a mini chart and top consumers right on a wall tablet — read-only, refreshed every minute. You can switch it off for dashboards in Energy Center's settings.

🛡️Security Center

Your whole home's security on one screen: every door, window, motion sensor, lock, person and camera — plus a simple PIN-protected alarm.

The board

  • Open doors/windows glow red, closed ones stay calm green; motion pulses; locks have inline Lock/Unlock buttons; people show 🏠/away; cameras show live snapshots (click = fullscreen). Updates every few seconds.
  • An event timeline records everything: opened, closed, locked, unlocked, arrived, left.

The alarm

  1. Create a PIN on first arm (big touch-friendly pad — works great on wall tablets).
  2. Arm Home, Away or Night. Arming Away starts the exit delay so you can walk out — sensors tripping during it are logged but never sound the alarm.
  3. When a monitored sensor trips while armed: a Delayed sensor starts the entry delay (type your code to cancel), an Instant sensor fires straight away.
  4. On trigger the alarm runs your Actions and alerts every channel you picked. Disarm any time with the PIN — that also stops the siren.

Choosing what it watches (Sensors tab)

  • Every door, window, motion sensor, lock and other binary sensor is listed — switch each one in or out of the system.
  • Per sensor: which modes watch it (Home / Away / Night) and Instant vs Delayed. Rule of thumb: the door you walk in through = Delayed, windows = Instant.
  • Entries marked auto are sensible defaults until you change them.

Actions & alerts

  • Actions tab: sound sirens, turn on lights/switches, lock doors, run scenes and scripts, take a camera snapshot, announce over speakers. A plain-English summary shows exactly what will happen.
  • Alerts tab: tick several notify services — phone push, HA notification, email, SMS — and use Send test alert to prove they work. Email needs an SMTP notify integration in Home Assistant; SMS needs a provider such as Twilio. Anything you add to HA appears in the list automatically.
  • Cameras tab: the selected camera's snapshot is attached to phone alerts, so you see the picture in the notification.

Keypad on a wall tablet

In Dashboard Maker, add the 🔐 Security keypad element (Home Life category — full keypad, compact shield or status bar). Mount the tablet by the door and anyone can arm or disarm with the PIN, no admin login needed. Turn it off any time with the "Allow the dashboard keypad" switch on the Status tab.

This is a smart-home alarm, not a certified security system: it works while HA and the add-on are running. Use it as a smart layer, not your only protection.

📋Family Board

The family organizer: shopping lists, chores with automatic rotation, and sticky notes. Built for wall tablets and fingers of all sizes.

🛒 Lists

Backed by Home Assistant to-do lists — so they sync with the HA mobile app and voice assistants. Add items with one tap, check them off with a satisfying strikethrough, clear completed in one go. ＋ New list creates a real HA list (Shopping, Hardware store, Packing…).

🧹 Chores

Each chore has an emoji, assignees and due days. The board shows whose turn it is today; Mark done advances the rotation and grows the 🔥 streak. The activity feed shows who did what, when.

“🗑 Trash out — Mon & Thu — Dad → Kid rotation” and “🍽 Dishes — every day — everyone in turn” ends the daily negotiation.

📌 Notes

Colored sticky notes for everything else — click to edit, autosaves. Every note has an audience: 🌍 Everyone, or 👤 specific users — a targeted note only appears on dashboards where that user is logged in. Anyone who sees a note can reply to it; the thread shows up both on the dashboards and here in the admin tool.

📺 On your dashboards

In the Dashboard Maker designer, the Home Life category has Family Notes and Shopping List widgets: notes appear as a sticky wall (tap one to read and reply), lists get big tap-to-check circles and an add row. The list widget is bound to a todo.* entity, which must be in that dashboard's entity allowlist.

Send “🦷 Dentist at 16:00, don't forget!” only to the kids' tablet — they tap it and reply “ok 👍”, and you see the reply in Family Board.

📢Announce & Intercom

Type a message, pick speakers, hit SEND — your house says it out loud (text-to-speech), optionally pinging phones too.

Using it

  1. Type the announcement (or tap a quick message chip — editable in settings: “Dinner is ready! 🍽”, “Leaving in 10 minutes 🚗”…).
  2. Tap speaker tiles to choose targets; each tile shows live state and has its own volume slider. Set default speakers once in settings.
  3. Optional: also send to phones as a notification.
  4. SEND. The history timeline lets you re-send anything with ↻.
No TTS engine? Add one free in Home Assistant: Settings → Devices & Services → Add integration → Google Translate text-to-speech. It appears here automatically.

📺 Intercom on your dashboards

The Intercom widget (designer → Home Life) puts a big announce button on any wall tablet. Tap it → pick an area (or Everywhere) → tap a quick message, type one, or hold the 🎤 to talk — your voice plays on the selected speakers like a real intercom.

Browsers only allow microphone recording on HTTPS pages. Tablets that open the panel over plain http://IP:8234 still get text and quick-message announcements — for voice, open the panel through your HTTPS domain. You can disable dashboard announcing entirely in this tool's settings.

🏠Away Simulator

On vacation, your lights follow your real routine: the simulator replays what each light did a week ago, shifted by a few random minutes — far more convincing than random flicker.

Setup

  1. Pick the lights/switches visible from outside. Use the ⚡ blink test to confirm each one responds.
  2. Settings: replay from 7 days back (same weekday — recommended), randomness ±15 min, optional active window, and pause when someone is home (it auto-pauses using person entities).
  3. Enable before you leave. Done.

Today's plan shows a 24-hour timeline per light — glowing segments for planned ON periods, ✓ marks for executed actions — and the activity log records everything it actually did.

It replays recorded history, so the lights you pick need a few days of history in Home Assistant (default retention is 10 days — “14 days back” may be empty on some installs).

❓FAQ & tips

Where is my data stored?

Everything lives in the add-on's /data folder (users, dashboards, schedules, alerts…) and survives updates and restarts. Helpers, automations and scenes are stored in Home Assistant itself.

Do the “live” tools need the add-on running?

Yes — Climate Scheduler, Security Center's alarm, Away Simulator and scheduled backups run inside the add-on. It starts automatically with Home Assistant (watchdog enabled).

How do I update?

Settings → Add-ons → Advance Tools → Update, like any add-on. Your data is kept. Changelog: the About page in the hub, or GitHub.

Something looks broken after an update

Hard-refresh the browser (Ctrl+F5) — tablets and browsers occasionally cache old files despite our no-cache headers.

A tool says “not connected to Home Assistant”

The add-on talks to HA over one shared WebSocket. Check that HA finished starting, then the add-on log (Settings → Add-ons → Advance Tools → Log).

Can normal (non-admin) users open tools?

No — tools and the hub are admin-only. Normal users only see the dashboards you granted them. That's what makes wall tablets safe.
